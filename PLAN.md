# Plan: Text-to-SQL Native Superset Extension

## Goal
Integrate the existing text2sql PoC directly into Apache Superset 5.x as a native extension.
Users type a natural language question in a right-sidebar panel inside SQL Lab and the generated
SQL appears directly in the editor — no Streamlit, no copy-paste.

## Architecture

```
bi-ai.text2sql/                         # extension root (new folder in bi-ai repo)
├── extension.json                      # metadata + contribution declarations
├── frontend/
│   ├── src/
│   │   ├── index.tsx                   # registers the SQL Lab right-sidebar view
│   │   ├── panel/Text2SqlPanel.tsx     # React panel: input + button + result
│   │   └── api/client.ts              # SupersetClient wrapper → backend endpoint
│   ├── package.json
│   ├── tsconfig.json
│   └── webpack.config.js              # Module Federation config (generated by CLI)
└── backend/
    ├── src/bi_ai/text2sql/
    │   ├── __init__.py
    │   ├── entrypoint.py              # import triggers @api registration
    │   ├── api.py                     # RestApi with POST /generate
    │   ├── llm.py                     # ported from PoC app/llm_client.py
    │   ├── prompt_builder.py          # ported from PoC app/prompt_builder.py
    │   ├── sql_validator.py           # ported from PoC app/sql_validator.py
    │   └── metadata.py               # uses Superset's Database model (not SQLAlchemy direct)
    └── pyproject.toml
```

The host Superset runs via Docker (apache/superset repo). The extension `.supx` is mounted
via a `docker-compose.override.yml` volume. A `superset_config_docker.py` enables the feature
flag and sets `EXTENSIONS_PATH`.

## Phase 1 — Backend Extension (Python)

**Goal:** A working `POST /extensions/bi-ai/text2sql/generate` endpoint inside Superset.

### Tasks

1.1 Create extension folder structure
```
bi-ai.text2sql/backend/src/bi_ai/text2sql/
```

1.2 Port PoC Python modules (minimal changes)
- `llm.py` ← `superset-text2sql-poc/app/llm_client.py`
  - Remove Streamlit imports
  - Read API key from `os.environ["LLM_API_KEY"]` (already done in PoC)
- `prompt_builder.py` ← `superset-text2sql-poc/app/prompt_builder.py`
  - Keep as-is; accepts schema dict, returns prompt strings
- `sql_validator.py` ← `superset-text2sql-poc/app/sql_validator.py`
  - Keep as-is; uses sqlglot (already a Superset dep)
- `metadata.py` ← NEW (replaces PoC's SQLAlchemy-direct version)
  - Use `from superset.models.core import Database` to get engine by `database_id`
  - Introspect tables/columns via `database.get_inspector()`

1.3 Write `api.py`
```python
from flask_appbuilder.api import expose, permission_name, protect, safe
from superset_core.rest_api.api import RestApi
from superset_core.rest_api.decorators import api
from .llm import generate_sql
from .metadata import describe_database

@api(id="text2sql", name="Text-to-SQL API")
class Text2SqlAPI(RestApi):
    @expose("/generate", methods=("POST",))
    @protect()
    @safe
    @permission_name("read")
    def generate(self):
        body = request.get_json(force=True)
        schema = describe_database(body["database_id"])
        result = generate_sql(body["question"], schema)
        return self.response(200, sql=result.sql, explanation=result.explanation)
```

1.4 Write `entrypoint.py`
```python
from . import api  # noqa: F401
```

1.5 Write `pyproject.toml`
- Dependencies: `apache-superset-core>=0.1.0rc2`, `openai`, `sqlglot`, `pyyaml`
- Do NOT include `apache-superset` as a dep (runtime-only host import)

1.6 Mount the backend into Docker
- Add `docker-compose.override.yml` at the Superset repo root:
```yaml
services:
  superset:
    volumes:
      - /Users/tommasoelia/Git/bi-ai/bi-ai.text2sql/dist:/app/extensions:ro
    environment:
      LLM_API_KEY: "${LLM_API_KEY}"
      LLM_MODEL: "${LLM_MODEL:-qwen/qwen3-coder}"
```

### Verification checklist
- [ ] `pytest backend/tests/test_api.py` passes (mocked LLM, real validator)
- [ ] `curl -X POST http://localhost:8088/extensions/bi-ai/text2sql/generate` without auth → 401
- [ ] Same with valid session cookie → returns `{"sql": "...", "explanation": "..."}`
- [ ] Generated SQL passes sqlglot parse for SQLite dialect

### Anti-patterns
- Do NOT auto-execute the SQL — return it as a string, never dispatch `runQuery`
- Do NOT put `LLM_API_KEY` in `extension.json` or any committed file

## Phase 2 — Frontend Extension (React/TypeScript)

**Goal:** A "Ask AI" panel in SQL Lab's right sidebar that calls the backend and injects SQL into the editor.

### Tasks

2.1 Scaffold frontend with extensions CLI
```bash
cd bi-ai.text2sql
superset-extensions init --frontend-only  # or use the full scaffold from Phase 0
```

2.2 Write `panel/Text2SqlPanel.tsx`
```tsx
import { sqlLab } from '@apache-superset/core';
import { generateSql } from '../api/client';

export function Text2SqlPanel() {
  const [question, setQuestion] = useState('');
  const [explanation, setExplanation] = useState('');
  const [loading, setLoading] = useState(false);

  async function handleGenerate() {
    setLoading(true);
    const tab = sqlLab.getCurrentTab();
    if (!tab) return toast.error('No active SQL Lab tab');
    const { sql, explanation } = await generateSql(question, tab.getDatabaseId());
    tab.getEditor().setValue(sql);   // fallback: dispatch queryEditorSetAndSaveSql
    setExplanation(explanation);
    setLoading(false);
  }

  return (
    <div style={{ padding: 16 }}>
      <h3>Ask AI</h3>
      <textarea value={question} onChange={e => setQuestion(e.target.value)} rows={4} />
      <button onClick={handleGenerate} disabled={loading}>
        {loading ? 'Generating…' : 'Generate SQL'}
      </button>
      {explanation && <p>{explanation}</p>}
    </div>
  );
}
```

2.3 Write `api/client.ts`
```ts
import { SupersetClient } from '@superset-ui/core';

export async function generateSql(question: string, databaseId: number) {
  const res = await SupersetClient.post({
    endpoint: '/extensions/bi-ai/text2sql/generate',
    jsonPayload: { question, database_id: databaseId },
  });
  return res.json as { sql: string; explanation: string };
}
```

2.4 Write `index.tsx` (registers the view)
```tsx
import { views } from '@apache-superset/core';
import { Text2SqlPanel } from './panel/Text2SqlPanel';

views.registerView(
  { id: 'bi-ai.text2sql.panel', name: 'Ask AI' },
  'sqllab.rightSidebar',
  () => <Text2SqlPanel />,
);
```

2.5 Build and bundle
```bash
cd bi-ai.text2sql/frontend
npm install
npm run build   # → dist/ contains the .supx
```

2.6 Point Superset at the dist folder (Phase 1 docker-compose.override.yml already mounts it)

### Verification checklist
- [ ] `npm test -- Text2SqlPanel` passes (SupersetClient and sqlLab mocked)
- [ ] Panel appears in SQL Lab right sidebar after `superset-extensions bundle` + host restart
- [ ] Typing a question and clicking "Generate SQL" injects SQL into the editor
- [ ] No auto-execution (SQL appears in editor, not in results pane)
- [ ] Anonymous (logged-out) user cannot call the backend (401 from `@protect()`)

### Anti-patterns
- Do NOT call OpenRouter directly from the browser (leaks API key)
- Do NOT dispatch `runQuery` — only set the editor value
- Do NOT edit the Module Federation `shared` block the CLI generated

## Key Decisions

| Decision | Choice | Reason |
|---|---|---|
| Integration path | Extensions system (SIP-177) | Official, no fork, works out-of-tree |
| Fallback if Extensions unavailable | `BLUEPRINTS` + React patch | Only if host can't run Superset 5.x |
| SQL injection | `tab.getEditor().setValue(sql)` → Redux fallback | Follow SDK shape; Redux if SDK unstable |
| LLM call location | Backend only | Keep API key server-side; reuse Superset RBAC |
| Auto-execution | Never | Carry PoC safety contract into the integration |
| Superset host | Docker (apache/superset master) | Standard dev environment |
